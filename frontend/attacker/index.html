ATT-01 MAVLink 명령 주입 (Unauthenticated Command Injection)
MAVLink v1/v2는 기본적으로 인증 메커니즘이 없다. FC(Flight Controller)와 같은 네트워크에 있는 누구든 TCP 소켓으로 직접 접속해 명령 패킷을 보낼 수 있다. 실전 전장에서는 재밍·스푸핑으로 C2 링크를 끊은 뒤 자체 송신기로 동일한 공격을 수행한다.
1공격 포트 :25760으로 TCP 직접 연결 (FastAPI 서버 완전 우회)
2HEARTBEAT 수신으로 드론 sysid·armed·mode 선확인
3COMMAND_LONG(msg_id=76) 바이너리 패킷 전송 — p2=21196으로 pre-arm 안전 체크 강제 우회
4COMMAND_ACK(msg_id=77) result 값으로 성공 판정
✔ 성공 기준 — COMMAND_ACK result=0 (ACCEPTED) + HEARTBEAT armed 비트 변화 실측
ATT-02 GNSS 스푸핑 (GPS Position Spoofing via MAVLink)
ArduCopter의 GPS_TYPE=14 설정을 이용하면 MAVLink 네트워크를 통해 외부에서 가짜 GPS 데이터를 FC에 직접 주입할 수 있다. EKF(Extended Kalman Filter)가 이 가짜 신호를 수용하면 드론은 실제로 잘못된 위치로 이동한다. 실전에서는 RF 스푸핑 장비로 동일한 효과를 낸다.
1PARAM_SET으로 GPS_TYPE=14 설정 — 외부 MAVLink GPS 소스 허용
2GLOBAL_POSITION_INT로 현재 실제 위치 읽기 (기준점 확보)
3GPS_INPUT(msg_id=113) 10Hz로 위·경도 오프셋을 단계적으로 증가시키며 주입
4EKF가 가짜 위치를 수용 → EKF_STATUS_REPORT flags 0x80(CONST_POS_MODE) 점등
✔ 성공 기준 — 주입 오프셋 누적 >5m + EKF flags 0x80 실측
⚠ 드론이 실제로 이동하므로 SITL 테스트 전용. 실기체 사용 금지
ATT-03 통신 두절 공격 (GCS Heartbeat Blackout → Failsafe RTL)
ArduCopter는 GCS로부터 FS_GCS_TIMEOUT(기본 5초) 동안 HEARTBEAT가 없으면 자동으로 RTL(Return To Launch) Failsafe를 발동한다. 공격자가 GCS로 위장해 HEARTBEAT를 전송한 뒤 갑자기 차단하면 드론은 임무 중 귀환을 강제당한다. 실전에서는 C2 주파수 재밍으로 동일 효과를 낸다.
1PARAM_SET으로 FS_GCS_ENABLE=1 확인·설정 (RTL Failsafe 활성화)
2GCS로 위장해 HEARTBEAT(MAV_TYPE_GCS) 1Hz 전송 → 드론이 GCS 연결 인식 (워밍업)
3★ HEARTBEAT 전송 즉시 중단 (재밍 시뮬레이션)
4타이머 경과 후 HEARTBEAT custom_mode 4(GUIDED) → 6(RTL) 전환 감지
✔ 성공 기준 — HEARTBEAT custom_mode=6(RTL) 전환 + 차단 후 경과 시간 실측
⚠ Armed+비행 중일 때만 Failsafe 발동. Disarmed 상태에선 STATUSTEXT 경고만 출력
RTL 후 GUIDED+GOTO 주입
